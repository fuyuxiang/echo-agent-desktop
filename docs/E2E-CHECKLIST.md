# Echo Agent 桌面客户端 — 端到端验收清单

本清单用于人工执行端到端验收。覆盖管理员组织管理、项目记忆组隔离、个人记忆本地隔离、模型配置下发以及断网降级。

按顺序执行，逐项勾选并核对预期结果。

---

## 前置准备

### 1. 启动 echo-agent-server

```bash
cd echo-agent-server
cp .env.example .env
# 编辑 .env，设置 ECHO_SERVER_SECRET 为任意非空字符串（客户端与服务端需一致约定）
npm run dev
```

预期：终端输出服务监听在 `http://localhost:8787`（或 `.env` 中配置的端口）。

- [ ] echo-agent-server 已启动，监听 8787，无报错

### 2. 启动桌面客户端

```bash
cd echo-agent-desktop
npm run dev
```

- [ ] 客户端窗口正常打开
- [ ] 完成 Onboarding 流程，本地 echo-agent 安装成功（首启会联网 pip 安装 Python 运行时，需保持网络畅通）
- [ ] Onboarding 完成后进入主界面

---

## 验收项

### 1. 管理员登录与组织管理

操作步骤：
1. 以初始管理员账号 `admin` 登录
2. 进入管理页，新建组 `g1`、`g2`
3. 在 `g1` 下新建一个成员账号 `a`
4. 在 `g2` 下新建一个成员账号 `b`

预期结果：两个组、两个成员均创建成功，管理页列表正确展示组与成员归属。

- [ ] admin 登录成功
- [ ] g1、g2 创建成功
- [ ] 成员 a 归属 g1，成员 b 归属 g2

### 2. 成员 a 触发项目记忆候选并共享到组

操作步骤：
1. 退出 admin，以成员 `a` 登录
2. 进入聊天页，进行一段会话，触发"项目记忆候选"（ShareMemoryDialog 弹窗）
3. 在弹窗中选择「共享到组」
4. 确认提交

预期结果：弹窗提交后提示「已共享到项目记忆」，该记忆写入 g1 的项目记忆。

> 注意：候选弹窗的自动触发依赖 echo-agent 下行信号，当前为联调待定项（见文末「已知联调待定项」）。验收前需确认触发链路已接通，或由联调脚本/手动注入候选触发弹窗。

- [ ] 聊天可正常收发
- [ ] 项目记忆候选弹窗出现
- [ ] 选择「共享到组」后提示共享成功

### 3. 成员 a 在记忆区看到该条项目记忆

操作步骤：
1. 成员 a 进入记忆区，切换到「项目记忆」Tab

预期结果：列表中出现第 2 步刚共享的那条项目记忆。

- [ ] 项目记忆 Tab 显示刚共享的记忆条目

### 4. 成员 b 的项目记忆为空（组隔离）

操作步骤：
1. 退出成员 a，以成员 `b`（g2）登录
2. 进入记忆区「项目记忆」Tab

预期结果：列表为空（或不含 g1 的记忆），证明项目记忆按组隔离。

- [ ] 成员 b 项目记忆 Tab 看不到 g1 的记忆

### 5. 成员 b 聊天检索不返回 g1 记忆

操作步骤：
1. 成员 b 在聊天页用与第 2 步相同的关键词进行检索/提问

预期结果：检索结果不包含 g1 的项目记忆，组隔离在检索链路同样生效。

- [ ] 成员 b 检索同关键词，结果不含 g1 记忆

### 6. 个人记忆本地隔离

操作步骤：
1. 成员 a 登录，在聊天/记忆中产生一条个人偏好记忆（个人记忆存本地）
2. 退出 a，成员 b 登录，查看个人记忆

预期结果：成员 b 看不到 a 的个人偏好，个人记忆按本地账号隔离，不上行服务器。

- [ ] 成员 a 个人偏好已保存在本地
- [ ] 成员 b 登录看不到 a 的个人记忆

### 7. 模型配置下发（allowLocalOverride=false）

操作步骤：
1. admin 登录，在管理页/模型配置中设置 `allowLocalOverride = false` 并下发
2. 退出 admin，成员登录，进入设置页查看模型配置表单

预期结果：成员设置页的模型配置表单为只读，无法本地覆盖管理员下发的配置。

- [ ] admin 设置 allowLocalOverride=false 成功
- [ ] 成员设置页模型配置表单只读、不可编辑

### 8. 断网降级

操作步骤：
1. 停止 echo-agent-server（或断开网络）
2. 在客户端聊天页继续对话

预期结果：聊天仍可用，本地推理正常；项目记忆注入降级为空（不阻塞、不报错崩溃），仅缺少服务端项目记忆上下文。

- [ ] 断开服务器后聊天仍可正常收发
- [ ] 项目记忆注入降级为空，无阻塞或崩溃
- [ ] 本地推理与个人记忆不受影响

## 已知联调待定项

- **记忆分流候选的来源协议（Task 7 留的 TODO）**：当前 WS 协议尚未约定"值得共享的项目记忆候选"的下行信号，即 echo-agent 何时、以何种消息格式下行候选尚待与 echo-agent 联调确定。
- **当前实现状态**：客户端 `ShareMemoryDialog`（共享到组/弃用的候选弹窗）与提交链路 `confirmShareToProject` 已就绪；聊天页保留了触发位（`src/renderer/src/pages/Chat/index.tsx` 中 `setCandidate(...)` 的接入点）。
- **联调动作**：协议确定后，在收到候选下行信号处调用 `setCandidate(candidate)` 即可触发弹窗，无需改动弹窗与提交逻辑。在此之前不臆造协议、不做假数据自动弹窗，验收第 2 步需借助联调链路或手动注入候选来触发。
